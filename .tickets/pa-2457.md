# Fix RLM fallback REPL input duplication and prompt race

In Bun fallback REPL, natural-language translation confirmation uses a second readline interface (`askInterface`) concurrently with main loop, causing doubled input handling and `RLM is processing a previous command` when answering prompts. Unify input handling so confirmation/ask prompts are handled by the same fallback line loop.



## Goal
Ensure fallback RLM input is single-stream and stable during translate-and-confirm prompts.

## Acceptance Criteria
- [ ] No secondary readline race in fallback path
- [ ] Answering 'Execute this code? [y/N]' does not trigger busy warning
- [ ] Single input line behaves normally (no duplicated processing)
