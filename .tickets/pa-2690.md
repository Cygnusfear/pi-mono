# Add hierarchical tracing for root and child RLM calls

Implement RLM trace logging for root and recursive child calls, including prompts, responses, code execution, budget events, and finalization. Expose trace path via CLI flag and provide run visibility for 30-minute scenarios.

Enable detailed monitoring of RLM execution tree across root and child calls with structured JSONL traces.

- [ ] Trace includes node start/end, llm prompt/response, code exec, budget updates, and final answer events
- [ ] Tracing is optional and has low overhead when disabled
